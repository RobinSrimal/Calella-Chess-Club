# D1 Infrastructure

## Purpose

D1 stores application state. Infra creates the database; `packages/db` owns schema and migrations.

## Resource

The implementation should create one D1 database for the application and link it to both Workers.

Expected binding name:

```txt
DB
```

## Migration Ownership

Migrations live under `packages/db`. Application code must not create tables at request time.

## Data Retention

Production D1 data must be retained. Non-production data can be removable.
